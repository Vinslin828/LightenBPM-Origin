# Ticket Info
GBPM-729: [Organization] Add `heads` field to GET /org-units API response

## Description
Background

The GET /org-units endpoint currently returns organization unit data without head information. Consumers need to make additional requests per unit to retrieve head data, which increases network overhead and complexity on the frontend.

User Story

As a frontend consumer of the GET /org-units API, I want the response to include a heads field for each org unit, so that I can display head information without making additional API calls.

Requirement

Add a heads field to each item in the GET /org-units response.

Current Response:

[
  {
    "id": 1,
    "code": "UNASSIGNED",
    "name": "Unassigned",
    "type": "ORG_UNIT",
    "children": [],
    "members": [],
    "createdAt": "2026-01-16T03:50:32.117Z",
    "updatedAt": "2026-01-16T03:50:32.117Z",
    "deletedAt": null
  }
]

Expected Response:

[
  {
    "id": 1,
    "code": "UNASSIGNED",
    "name": "Unassigned",
    "type": "ORG_UNIT",
    "children": [],
    "members": [],
    "heads": [],
    "createdAt": "2026-01-16T03:50:32.117Z",
    "updatedAt": "2026-01-16T03:50:32.117Z",
    "deletedAt": null
  }
]

API Reference

Request:
GET https://bpm.dev.lightencycling.com/org-units?filter=ORG_UNIT
Authorization: Bearer eyJraWQiOi...PmoJA

[
  {

Acceptance Criteria

[ ] GET /org-units response includes a heads field on each org unit object
[ ] heads is an array (empty array [] when no heads are assigned)
[ ] Each item in heads follows the same head schema used by GET /org-units/{id}
[ ] Existing fields are not affected
