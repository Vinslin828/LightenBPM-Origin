# Ticket Info
GBPM-721: User defaultOrgId and defaultOrgCode are inconsistent between GET /bpm/users and GET /bpm/org-units/{id} member data

## Description
[Pre-conditions]

User ramble_user_003 (id: 70) exists with defaultOrgId: 40 and defaultOrgCode: "ram003".

Org unit ram003 (id: 40) exists and has user ramble_user_003 as a member.

[How to reproduce the problem]

Call GET /bpm/users and note user id 70 (ramble_user_003) returns defaultOrgId: 40 and defaultOrgCode: "ram003".

Call GET /bpm/org-units/40 to fetch the org unit that the user belongs to.

Inspect the members array in the response and locate the entry for user id 70.

Verify the defaultOrgId and defaultOrgCode fields on the member object.

[Actual Result]

The member object inside GET /bpm/org-units/40 returns defaultOrgId: 0 and defaultOrgCode: "" for user id 70.

[Expected Result]

The member object inside GET /bpm/org-units/40 should return defaultOrgId: 40 and defaultOrgCode: "ram003", consistent with GET /bpm/users.
