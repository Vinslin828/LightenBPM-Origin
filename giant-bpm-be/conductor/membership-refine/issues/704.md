# Ticket Info
GBPM-704: [User Management] Unable to change default organization when user has two orgs assigned with overlapping date ranges

## Description
[Pre-conditions]

On 'Users'.

[How to reproduce the problem]

Create a new user with a default org-Org A and default start date/end date.

Click the edit icon in the Organizations section to open the Organization modal.

Re-open the Organization modal and select Org B with the same date range as Org A, then click OK. → Verify: Both Org A and Org B appear in the Organizations section.

Re-open the Organization modal and click "Set as default" on Org B, then click OK.
→ Observe whether the default org changes from Org A to Org B.

From the Organizations section in User Details, click "Set as default" next to Org B.
→ Observe whether the default org changes.

[Actual Result]
When a user is assigned to two organizations with overlapping (identical) date ranges, the default organization cannot be changed. Clicking "Set as default" on Org B in either the Organization modal or the Organizations section has no effect — the default remains unchanged and the UI does not reflect any update.

[Expected Result]
The admin should be able to change the default organization at any time regardless of the date ranges assigned to the organizations. Clicking "Set as default" on any active organization should update the default org and reflect it immediately in both the Organization modal and the Organizations section in User Details.

# Fixing Ideas
I think the root cause should be how we treat the memberships. We need to dive into the logic and figure out does following concpets meet expectation:
1. Default Org of User: Idealy, the only org that user has active membership should be user's default org. If user have multiple active membership we need to follow the default org preference implementation in user.services
2. We need to review the revised overlapping detection logic and the user api implementations for any possible conflicts
